Curriculum

4 Modules Across 30 Hours

01

AI Fundamentals and Literacy

Think. Understand. Apply.

8 MCQs
  • History and evolution of AI — from rule-based systems to large language models
  • Types of AI — narrow AI, general AI, supervised, unsupervised, and reinforcement learning
  • How machine learning models are trained — data, parameters, weights, and loss functions explained simply
  • AI vs automation vs intelligence — where the line is and why it matters for careers
  • Real-world AI applications across industries — healthcare, finance, logistics, education, and creative fields
  • Ethical considerations in AI — bias, hallucination, data privacy, and responsible use

Module Task

AI Scenario Analysis — Students are given 5 real-world scenarios across different industries. They must classify the AI type involved, identify the data likely used to train it, and flag one ethical risk per scenario. Written submission.

02

Generative AI

Create. Prompt. Evaluate.

8 MCQs
  • What generative AI is and how it differs from traditional AI — transformers, diffusion models, and foundation models explained
  • How large language models work — tokens, context windows, temperature, and inference
  • Prompt engineering fundamentals — zero-shot, few-shot, chain-of-thought, and role prompting
  • Multimodal AI — text to image, text to audio, text to video, and code generation
  • Limitations of GenAI — hallucination, context loss, bias, and when not to use it
  • GenAI use cases by domain — research, content creation, software development, data analysis, and education

Module Task

Personal Prompt Sequence — Students pick one domain relevant to their discipline and build a 5-prompt sequence using at least 3 different prompting techniques. They must document what changed in output quality between each technique and explain why.

Tools: GPT-4, Gemini, Midjourney, Runway, Suno, GitHub Copilot

03

Claude Masterclass

Build. Delegate. Automate.

12 MCQs
  • Part A — Claude Foundations: What makes Claude different — memory architecture, safety design, context handling, and agentic capability. The 3 Claude products: Chat, Cowork, and Code. Model selection: Haiku 4.5, Sonnet 4.6, Opus 4.6.
  • Part B — Memory & Personalization: The 3 memory layers — Global Instructions, Project-level claude.md files, and Skills. The Brain Imprint method.
  • Part C — Skills: The TAR Framework (Trigger, Action, Result). Anatomy of a skill folder. Writing skill descriptions that fire precisely. Building your first skill end to end.
  • Part D — MCP Servers & Connectors: What MCP is and why it changes how Claude works. Connecting Claude to Google Drive, Gmail, Google Calendar, Notion, and Slack. Building one MCP-connected workflow end to end.

Module Task

Build a Skill + MCP Workflow — Students build one complete skill using the TAR framework for a use case relevant to their own discipline. They then connect Claude to one MCP tool, run a live prompt against real data, and document the before and after. Screenshot evidence required.

Tools: Claude Chat, Claude Cowork, Claude Code, Whisperflow, MCP Connectors

04

Agentic AI

Design Multi-Step Workflows End to End

10 MCQs
  • Part A — Agentic AI Foundations: The shift from single-turn responses to multi-step autonomous execution. How AI agents work — perception, planning, memory, tool use, and action loops. Types of agents and the decision framework.
  • Part B — Workflow Automation with n8n: n8n interface walkthrough — nodes, workflows, credentials, and execution logs. Connecting n8n to Claude via API. Building a multi-step agentic workflow: web scrape trigger → Claude analysis → Slack notification → Google Sheets logging.
  • Part C — Capstone Workflow Build: Students design and build one complete agentic workflow. Workflow must include a trigger, at least 3 connected nodes, a Claude processing step, and a final output destination. Peer walkthrough with explanation and time savings estimate.

Module Task

Build a Live Agentic Workflow — Build one fully functional agentic workflow end to end using n8n. Workflow must include Claude as a processing node. Students submit a screen recording of the workflow running live, a written explanation of what it automates, and an estimate of time saved per week.

Tools: n8n, Claude API, Google Sheets, Gmail, Slack, Webhooks
